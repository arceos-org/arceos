# ArceOS 新手GitHub贡献一键操作指南
## 适用人群
无GitHub开源贡献经验、想为ArceOS做文档/代码贡献的入门者
## 核心说明
本文档为纯新手向，全程步骤式操作，教你快速完成「Fork仓库→新建文件→提交PR」全流程，无需任何开源经验和代码基础。
## 一键操作步骤（全程点鼠标，无复杂指令）
### 步骤1：Fork ArceOS官方仓库
1. 打开ArceOS官方仓库：https://github.com/arceos-org/arceos
2. 找到页面**右上角**的「Fork」按钮，点击
3. 等待跳转后，直接点击「Create fork」，完成仓库复刻（不用改任何设置）

### 步骤2：在自己的仓库新建文件
1. 进入你Fork后的ArceOS仓库（页面顶部显示你的账号名+arceos）
2. 找到页面**中间**的「Add file」下拉按钮，点击选择「Create new file」
3. 页面跳转到新建文件页，仅需2步：
   - 顶部输入**文件路径/文件名**（按社区规范填写，如docs/xxx-zh.md）
   - 中间编辑区粘贴你要贡献的内容（文档/注释均可）

### 步骤3：保存新建的文件
1. 拉到页面**最底部**，找到「Commit new file」按钮
2. 不用改任何文字，直接点击按钮，完成文件保存（新手默认设置即可）

### 步骤4：提交PR（贡献成功的核心步骤）
1. 回到你Fork后的仓库首页，找到页面**上方**的「Contribute」下拉按钮，点击
2. 选择「Open pull request」，跳转到PR提交页
3. 不用改标题/描述（系统会自动显示你新建的文件），直接拉到页面底部点击「Create pull request」
4. 页面跳转后，你的PR就提交成功了，复制当前页面的链接，就是你的开源贡献证明！

## 新手避坑2点提醒
1. 全程不用修改官方仓库的任何文件，仅在自己Fork的仓库中操作
2. 提交PR后无需额外操作，社区会自动审核，新手的文档类贡献几乎100%通过
